# Support Playbook - Dominican Republic Pilot

Before go-live, confirm the support WhatsApp line shown in the app and ops diagnostics is the real Dominican support number. Do not operate with placeholder numbers.

## Customer support WhatsApp handling

Collect these first:

- Customer name
- Phone number
- Order number
- Active city
- Restaurant name
- Payment method

## Common issues and first responses

### Order not received

- Check tracking status.
- Confirm whether a driver is assigned.
- If the order is delayed, send the latest ETA.
- Escalate to ops if the order is stuck without driver action.

### Wrong address

- Confirm the address stored on the order.
- If the order is still preparing, coordinate with restaurant and dispatch.
- If already out for delivery, escalate to ops immediately.

### Payment confusion

- Confirm payment method and current payment status.
- Do not mark cash orders as paid without confirmation from finance or ops.
- Escalate suspicious duplicate-payment claims.

### Promo code problems

- Ask for the promo code used and screenshot if possible.
- Confirm the city and restaurant.
- If the code is invalid or expired, explain the rule clearly.

### Referral reward delay

- Explain that referral reward is applied after successful delivered completion.
- Confirm the order was fully delivered and finalized.
- Escalate if more than one business day passes after finalization.

### Driver late or merchant slow

- Check whether the delay is pre-dispatch, merchant acceptance, or in-transit.
- Escalate merchant-side delays to restaurant support.
- Escalate in-transit delays to ops.

## When to escalate to ops

- Driver unavailable or wrong-city assignment risk
- Order stuck without status changes
- Repeated cancellation pattern
- ETA breach with active customer complaint

## When to refund manually

- Customer charged incorrectly and finance confirms
- Order cannot be fulfilled and cancellation is approved
- Promo or referral issue requires manual goodwill credit

## Data support should request

- Order number
- Customer phone
- City
- Restaurant name
- Payment method
- Screenshot of error or payment proof if relevant

## Order history note

- Customer order history is no longer phone-only.
- Ask the customer to open the Orders tab from the same device used at checkout so the verified history flow can complete.
- If the customer changed devices, explain that they need a recent verified checkout session on the current device before history will load.
